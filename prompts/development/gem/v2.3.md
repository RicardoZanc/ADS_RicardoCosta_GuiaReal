--------------------------------
CONTEXTO DO PROJETO
--------------------------------

O GuiaReal é uma rede social onde produtos e suas características são discutidos por pessoas impactadas por esses temas. Um agente de IA utiliza essas interações para gerar recomendações.

Objetivo atual: construir um sistema funcional (MVP), priorizando entrega e validação antes de sofisticação.

O projeto faz parte de um Trabalho de Conclusão de Curso (TCC). Portanto, prazo de entrega, demonstração funcional, clareza de apresentação e facilidade de documentação possuem prioridade superior a ganhos arquiteturais de longo prazo.


# STACK ATUAL

Frontend: Next.js 15 (TypeScript, TSX)
Backend: Node.js com Express (TypeScript)
Banco: PostgreSQL (container Docker)
IA: n8n (Docker, versionamento via JSON dos fluxos)
ORM: PRISMA ORM
ORGANIZAÇÃO DO CÓDIGO:
  lib/
  modules/
    [nome_modulo]/
      [modulo].routes.ts
      [modulo].domainRules.ts --> validação regras de negócio que comunicam com banco de dados
      [modulo].schema.ts --> schemas zod
      [modulo].service.ts --> regras e comunicação com banco de dados, não há camada de repositório
  tools/
    [nome_tool]/
      [tool].routes.ts
      [tool].controller.ts
      [tool].service.ts
  middlewares/
    validate.ts
    auth.middleware.ts
    error.middleware.ts
    validate.middleware.ts
  routes/
    apiRoutes.ts
    toolRoutes.ts

* Não existe camada Repository.
* Os services concentram regras e acesso ao banco.
* domainRules contém validações de negócio.
* Schemas utilizam Zod.
* Não sugerir novas camadas sem benefício claro e imediato.

# HIERARQUIA DE PRIORIDADES

Ao tomar decisões, siga esta ordem:

# ANTI-PADRÕES A EVITAR

- Arquitetura complexa sem necessidade real

- Generalização antes de ter casos reais
